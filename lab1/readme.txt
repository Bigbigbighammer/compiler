【项目概述】
================================================================================
本项目是一个基于Java的个人所得税计算系统，实现了一个完整的面向对象程序设计示例。

主要功能：
  ✓ 根据用户输入的月工资计算应缴纳的个人所得税
  ✓ 支持从JSON配置文件灵活加载税率表
  ✓ 支持动态修改起征点（免税额）
  ✓ 支持重置税率表或仅调整税率
  ✓ 提供友好的控制台交互菜单
  ✓ 完整的数据验证和异常处理
  ✓ 详细的javadoc和设计文档

【技术特点】
================================================================================
编程语言: Java 8+
设计模式: 责任链模式、工厂模式、DTO模式
架构风格: 分层架构（表现层-业务逻辑层-数据访问层）
配置管理: JSON格式，Gson库解析
版本控制: Git

核心设计亮点：
  ✓ 使用责任链模式实现阶梯税率计算
  ✓ 清晰的分层架构设计
  ✓ 完整的单一职责原则应用
  ✓ 灵活的扩展性和可维护性
  ✓ 专业级的文档和注释

【使用方法】
================================================================================

1. 编译项目
   cd D:\Project4J\compiler\lab1
   javac -encoding UTF-8 -cp "lib/gson-2.10.1.jar" -d bin \
     src/model/entity/*.java \
     src/model/loader/*.java \
     src/service/*.java \
     src/view/*.java \
     src/Main.java

2. 运行程序
   java -cp "lib/gson-2.10.1.jar;bin;." Main

3. 程序菜单
   当前起征点: 1600 元
   请选择功能:
     1. 计算个人所得税
     2. 设置起征点
     3. 重置整个税率表 (重新录入区间和税率)
     4. 仅调整税率 (保留区间不变)
     0. 退出

4. 示例计算
   月工资: 2000元  →  应缴税: 20元
   月工资: 3000元  →  应缴税: 115元
   月工资: 5000元  →  应缴税: 385元

【项目收获体会】
================================================================================

本项目的开发过程中，收获颇丰：

【一】面向对象设计
  ✓ 深刻理解了对象的封装、继承、多态等概念
  ✓ 学会了如何设计清晰的类结构和接口
  ✓ 体会到了单一职责原则的重要性
  ✓ 实践了如何通过继承和接口实现代码复用

【二】设计模式应用
  ✓ 责任链模式: 用于实现阶梯税率的逐级计算
    - 清晰地分离了不同等级的计算逻辑
    - 使得新增税率等级变得简单
    - 这是我第一次在真实项目中应用设计模式

  ✓ 工厂模式: 用于创建不同类型的配置加载器
    - 隐藏了对象创建的细节
    - 为未来的扩展奠定了基础
    - 提高了代码的可维护性

  ✓ DTO模式: 用于JSON和Java对象的数据转换
    - 清晰地定义了数据结构
    - 便于与Gson库的集成
    - 体会到了良好的数据结构设计的重要性

【三】分层架构
